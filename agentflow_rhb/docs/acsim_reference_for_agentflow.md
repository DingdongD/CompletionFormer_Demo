# ACSim Docs As AgentFlow Reference

## Verdict

`/root/demo/ACSim/docs` is useful as a design reference for AgentFlow, but not
as a drop-in runtime for the current RHBLite deployment path. It gives concrete
contracts for memory layout, tensor tiling, compiler handoff, validation gates,
and package-quality checks. Those concepts map directly to gaps in AgentFlow:
layout-aware submodel scoring, legality checks, generated package validation,
and schedule-quality diagnostics.

## Useful Concepts

### Memory And Tensor Layout

Relevant docs:

```text
/root/demo/ACSim/docs/memory_layout.md
/root/demo/ACSim/docs/tensor_layout.md
```

Key points to reuse:

- Explicit memory hierarchy: external bank groups plus local buffers.
- Buffer roles: feature maps in LBUF, weights/matrices in RBUF, parameters and
  masks/scales in CWBUF.
- Row widths and capacities should be first-class constraints.
- Tensor layouts are defined with tile mode, data type, padding shape, storage
  order, and pack flags.
- Non-divisible shapes are padded according to memory-row/tile shape, not by
  ad-hoc model-level convenience.

AgentFlow implication:

```text
ONNX node/subgraph -> layout plan -> tile/pad plan -> memory-fit score
```

This can improve the current black-box split policy. Instead of only learning
from pass/fail, AgentFlow can predict risky shapes before compile/board tests.

### Co-Validation Pattern

Relevant docs:

```text
/root/demo/ACSim/docs/tapu/integration_overview.md
/root/demo/ACSim/docs/tapu/TAPU_COVALIDATION_CASES.md
```

Key points to reuse:

- Every package runs against a reference result.
- Static synthetic operator suites and small real-model suites are separated.
- Layout audit compares fast path and cycle-accurate path with the same visible
  mask.
- Validation is not only numerical; generated instruction shape and schedule
  quality are also guarded.

AgentFlow implication:

```text
candidate subgraph
  -> compile/cmodel
  -> board
  -> numerical check
  -> artifact-quality check
  -> rule update
```

This matches our current workflow, but ACSim makes it more systematic. We
should add explicit `quality gates` to AgentFlow reports:

- instruction count
- DMA/load/store/fence count
- loop presence for repeated bodies
- buffer fit and padding overhead
- expected versus observed latency

### Compiler Handoff Contract

Relevant docs:

```text
/root/demo/ACSim/docs/tapu/TAPU_MORPH_CWBUF_HANDOFF.md
/root/demo/ACSim/docs/tapu/global_instruction_contract.md
/root/demo/ACSim/docs/tapu/micro_instruction_and_cwbuf.md
```

Key points to reuse:

- Compiler output is treated as hardware-visible payload, not a semantic blob.
- Global instruction/package metadata is generated by the simulator/package
  owner, not hidden inside compiler output.
- Input/output movement must be explicit; no hidden DMA wrapping.
- Dynamic templates reserve and patch concrete shape instances.
- The loader path is contractual: compiler payload, package layout, memory
  names, and execution must agree.

AgentFlow implication:

AgentFlow should model a deployment package as a contract with explicit fields:

```text
submodel id
input buffers
output buffers
layout/tile mode
quant scale
compiler artifacts
packer artifacts
DMA/copy/glue steps
reference tensors
board result
latency counters
```

This would make host/RHB scheduling more reproducible and easier to debug than
the current script-level glue.

### Dynamic Shape And Template Ideas

ACSim's dynamic validation uses compiler templates plus concrete shape packing.
That is relevant to AgentFlow for models with repeated shapes or runtime-fixed
but model-variable sizes.

Possible AgentFlow use:

- Cache compiled submodel templates by operator family and layout class.
- Specialize templates for concrete shape during deployment.
- Reuse known-good patterns for repeated blocks, such as ResNet BasicBlock,
  decoder Conv blocks, and CSPN depth/guidance heads.

This is especially relevant for our split-conv and host-sample plus RHB-1x1
patterns.

## What Does Not Directly Transfer

- ACSim TAPU is BF16 SIMD-oriented; our current RHBLite flow is ONNX compile,
  packer, int8-ish deployment, `ac_driver`, and board log comparison.
- ACSim memory names and TSX package files are TAPU-specific.
- Morph-MLIR TAPU CWBUF flow is not the same as the RHBLite packer output.
- We should not assume ACSim-supported ops are RHBLite-supported ops.

So the transfer should be conceptual and rule-driven, not a direct execution
replacement.

## Concrete AgentFlow Enhancements

### 1. Layout Analyzer

Add an analyzer that annotates each tensor/subgraph with:

```text
layout: HWC / XY / CoCiKhKw
tile_mode candidate
data_width
padding_shape
padded_shape
padding_overhead
estimated local buffer bytes
estimated weight buffer bytes
```

This should feed the submodel scorer before compile.

### 2. Shape Risk Rules

Use ACSim-style tiling/padding vocabulary to formalize rules we discovered
empirically:

- channel dimensions should prefer multiples of tile groups.
- high-channel small-spatial Conv is risky and should trigger split candidates.
- stride/downsample Conv should trigger exact offset split or retrainable
  sample+1x1 candidates.
- concat/add/residual should be explicit Host glue unless a whole block has
  been validated.

### 3. Package Contract Manifest

Every generated deployment package should include:

```json
{
  "submodel": "...",
  "inputs": [],
  "outputs": [],
  "host_glue_before": [],
  "host_glue_after": [],
  "layout": {},
  "quant": {},
  "compile": {},
  "packer": {},
  "validation": {},
  "latency": {}
}
```

This mirrors ACSim's emphasis on package-visible contracts.

### 4. Validation Suites

Add two AgentFlow suites:

- `synthetic-verify`: small operator/layout probes generated from rule DB.
- `real-model-verify`: representative subgraphs from CompletionFormer, CSPN,
  and Depth-Anything.

This matches ACSim's simple/real split and would prevent regression when rules
change.

### 5. Artifact Quality Checks

Beyond `All same`, parse and score:

- instruction count
- params size
- output address and buffer placement
- timeout markers
- stale output markers
- launch count
- latency counters

This is the AgentFlow equivalent of ACSim's instruction/schedule-quality guard.

## Recommended Next Implementation Order

1. Implement a small `layout_model.py` in AgentFlow with tile/padding estimators.
2. Extend ONNX graph importer annotation with padded-shape and buffer estimates.
3. Add package manifest generation for each candidate submodel.
4. Add synthetic verification cases generated from the rule DB.
5. Add quality scoring into deep search: minimize host/RHB transfers, launch
   count, padding overhead, split count, and observed board latency.

## Bottom Line

ACSim docs are a strong reference for making AgentFlow less ad-hoc. The most
valuable transfer is not TAPU-specific code, but the discipline:

```text
layout-aware legality -> explicit package contract -> reference co-validation
-> schedule/artifact quality checks -> rule updates
```

This should become the next layer of AgentFlow automation before we try to
deploy more complex models end to end.
